gitx — AI-powered Git assistant

Generate commit messages, describe repository state, and create changelogs — all from your terminal, powered by AI.

Go 1.26+ Linux / macOS / Windows OpenAI · DeepSeek · Anthropic · Google · Ollama

Quick Start

# Install
curl -fsSL https://raw.githubusercontent.com/user/gitx/main/scripts/install.sh | sh

# Run setup (picks provider, model, API key)
gitx setup

# Generate a commit message
git add .
gitx commit

# Describe your current branch
gitx describe --base main

# Generate a changelog
gitx changelog --latest

Commands

gitx commit — Generate a commit message

Analyze changes and generate a conventional commit message with interactive confirmation.

# Auto: staged first, unstaged fallback
gitx commit

# Staged changes only
gitx commit --staged

# Unstaged changes only
gitx commit --unstaged

# Split changes by directory into separate commits
gitx commit --group
FlagDescription
--stagedUse only staged changes
--unstagedUse only unstaged changes (unstage everything first, then stage unstaged)
--groupSplit changes by directory into separate commits
--providerOverride AI provider
--modelOverride AI model

Interactive flow: After generation, you're prompted with Y (commit), N (cancel), E (edit in $EDITOR), R (regenerate, up to 3 attempts), or C (copy to clipboard).

gitx describe — Describe repository state

Describe the current state of your repository — commits, staged changes, and unstaged changes.

# Last 10 commits
gitx describe

# Last 5 commits
gitx describe --commits 5

# All commits since main
gitx describe --base main

# Unstaged changes only
gitx describe --unstaged

# Staged changes only
gitx describe --staged

# Full picture (staged + unstaged)
gitx describe --staged --unstaged

# Write to a file
gitx describe --output state.md
FlagDescription
-c, --commitsNumber of recent commits to include (default 10)
-s, --stagedInclude staged changes (excludes commits by default)
-u, --unstagedInclude unstaged changes (excludes commits by default)
--baseBase branch for commit comparison (shows all commits since this branch)
-o, --outputWrite output to a file

gitx changelog — Generate changelog entries

Analyze git tags and commits to generate structured changelog entries.

# Latest tag only
gitx changelog --latest

# From a tag to HEAD
gitx changelog --from v1.0.0

# Range between tags
gitx changelog --from v1.0.0 --to v2.0.0

# Write to file
gitx changelog --output CHANGELOG.md
FlagDescription
--latestGenerate for the latest tag only
--fromStart version tag (inclusive)
--toEnd version tag (inclusive, defaults to HEAD)
-o, --outputWrite output to a file

gitx config

Manage configuration — list, get, and set settings like provider, model, and API key.

gitx setup

Interactive setup wizard that walks you through choosing a provider, model, and entering your API key.

gitx doctor

Diagnose installation and configuration — checks git, config, API keys, and clipboard tools.

Configuration

GitX uses a single config file at ~/.config/gitx/config.yaml:

ai:
  provider: deepseek
  model: deepseek-v4-flash
  api_key: sk-...
commit:
  style: conventional       # conventional or gitmoji

Architecture

GitX uses a layered architecture with clear separation of concerns. Each layer has a single responsibility and communicates through well-defined interfaces.

CLI Layer       gitx commit | describe | changelog | setup | config | doctor
                        │
Services Layer    CommitService | DescribeService | ChangelogService
                        │
Git Layer         Client interface → ExecClient (git via os/exec)
AI Layer          Provider interface → OpenAI | DeepSeek | Anthropic | Google | Ollama
Prompts Layer     Builder[T] → CommitBuilder | DescribeBuilder | ChangelogBuilder
                        │
Security          Secret scanner (redacts keys/tokens before sending to AI)
Config            ~/.config/gitx/config.yaml
UI                Spinner · Interactive prompts · Clipboard · Editor

Git Aliases

GitX ships with a set of recommended git aliases for daily use. Install them optionally:

sh scripts/git-aliases.sh --install

Or add them manually with git config --global:

# gitx aliases
[alias]
  gbc  = branch --show-current
  gbd  = branch -D
  gc   = checkout
  gcn  = checkout -b
  glo  = log --oneline --graph
  gpc  = push -u origin HEAD
  gpcf = push --force-with-lease origin HEAD
AliasCommandDescription
gbcgit branch --show-currentShow current branch name
gbdgit branch -D <branch>Delete a branch
gcgit checkout <branch>Checkout existing branch
gcngit checkout -b <branch>Checkout new branch
glogit log --oneline --graphShow compact commit log
gpcgit push -u origin HEADPush current branch (set upstream)
gpcfgit push --force-with-lease origin HEADForce push current branch safely

Uninstall

To completely remove gitx from your system, follow the steps for your platform.

macOS / Linux

1. Remove the binary

# If installed without sudo (default location)
rm -f ~/.local/bin/gitx

# If installed with sudo
rm -f /usr/local/bin/gitx

# If you specified a custom GITX_INSTALL_DIR
rm -f "$GITX_INSTALL_DIR/gitx"

2. Remove the config file

rm -rf ~/.config/gitx

3. Remove git aliases

sh scripts/git-aliases.sh --uninstall

Or remove them manually:

git config --global --unset alias.gbc
git config --global --unset alias.gbd
git config --global --unset alias.gc
git config --global --unset alias.gcn
git config --global --unset alias.glo
git config --global --unset alias.gpc
git config --global --unset alias.gpcf

4. Verify removal

# Should print "command not found"
gitx

# Check for leftover config
ls ~/.config/gitx   # should show "No such file or directory"

Windows (PowerShell)

1. Remove the binary

# Find where gitx is installed
where.exe gitx

# Remove it (adjust path as needed)
Remove-Item C:\Users\<user>\bin\gitx.exe

2. Remove the config file

Remove-Item -Recurse -Force $env:USERPROFILE\.config\gitx

3. Remove git aliases

git config --global --unset alias.gbc
git config --global --unset alias.gbd
git config --global --unset alias.gc
git config --global --unset alias.gcn
git config --global --unset alias.glo
git config --global --unset alias.gpc
git config --global --unset alias.gpcf

4. Verify removal

# Should print an error
gitx --version

# Check for leftover config
Test-Path $env:USERPROFILE\.config\gitx   # should show False

One-liner (macOS / Linux)

# If installed without sudo
rm -f ~/.local/bin/gitx && rm -rf ~/.config/gitx && for a in gbc gbd gc gcn glo gpc gpcf; do git config --global --unset alias.$a 2>/dev/null; done

# If installed with sudo
rm -f /usr/local/bin/gitx && rm -rf ~/.config/gitx && for a in gbc gbd gc gcn glo gpc gpcf; do git config --global --unset alias.$a 2>/dev/null; done
